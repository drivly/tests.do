How to use

Tests.do is a simple tool to create, and manage your tests all within a single URL.

All you need to do to get started is think up what you want to test, and write a small bit of config in the URL and you are done:

./status == 200&headers.content-type += image/bucket.do/read/file.png Will test:
  • The status code of the URL is 200
  • The content-type header contains the string "image"
Rules

Tests.do uses a simple, yet powerful rule syntax to create your tests.

The rules are split into two parts:

  • The path to the property you want to check
  • The operator and value you want to check against

The path is a series of dot-separated properties, that are traversed down the response object.

The operators are simple strings that are used to compare the value of the property against the value you specify.

The value is simply what you want to check against.

Operators

The following operators are available:

  • == - The property must be equal to the value
  • != - The property must not be equal to the value
  • += - The property must include the value
  • -= - The property must not include the value
  • >= - The property must be greater than the value
  • <= - The property must be less than the value
